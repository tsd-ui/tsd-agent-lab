# Branch-Only Mode

## What it means

In `branch-only` mode the agent may read and edit files and run tests, but it
must not commit or push. The harness runs the agent in an isolated git
worktree. When the run completes, `create-local-branch.sh` stages all
uncommitted changes, creates a branch named `agent-lab/<task_id>`, and commits
with a descriptive message — but does **not** push.

You inspect the branch, adjust if needed, and push manually when satisfied.

## When to use branch-only

| Use branch-only when… | Use patch-only when… |
|----------------------|-----------------------|
| You want a clean commit ready to push | You want to review a raw diff before committing |
| The change is ready for a draft PR | The change is exploratory or uncertain |
| You want a tidy git history | You plan to cherry-pick hunks manually |
| You prefer `git push` and a PR | You prefer applying via `git apply` |

See [patch-only-mode.md](patch-only-mode.md) for the patch export workflow.

## Running a branch-only task

```bash
# 1. Create a run directory from your task file
run_dir=$(./harness/create-run.sh examples/tasks/pilot-branch-only.yaml)

# 2. Clone the repo and create the worktree
./harness/prepare-repo.sh "$run_dir"

# 3. Run the agent
./harness/run-claude.sh "$run_dir"

# 4. (Optional) Verify outputs
./harness/verify-run.sh "$run_dir"

# 5. Create the local branch
./harness/create-local-branch.sh "$run_dir"
```

The script creates a branch named `agent-lab/<task_id>` inside the worktree,
staged and committed with:

```
feat(agent-lab): <title>

Run ID: <run_id>
Mode: branch-only
Task: <task_id>

Generated by TSD Agent Lab harness. Human review required before push.
```

## Reviewing and pushing

```bash
# Inspect the branch
cd <worktree>
git log --oneline
git diff HEAD~1

# Push when ready
git push -u origin agent-lab/<task_id>

# Open a draft PR (GitHub CLI)
gh pr create --draft --title "<title>" --head agent-lab/<task_id>
```

## Force flag

If a branch with the same name already exists (e.g., from a previous run of
the same task), the script refuses to overwrite it by default:

```bash
# Overwrite an existing branch
./harness/create-local-branch.sh "$run_dir" --force
```

`--force` deletes the existing local branch and recreates it. It does not
affect any remote tracking branch.

## Dry-run

```bash
./harness/create-local-branch.sh "$run_dir" --dry-run
```

Prints exactly what would be staged, committed, and pushed — no changes made.

## Failure modes

**No changes found.** The script exits cleanly with a warning. Possible
causes: the agent made no edits, or it committed changes already (check
`git log` in the worktree).

**Branch already exists.** Use `--force` to overwrite, or delete the branch
manually: `git branch -D agent-lab/<task_id>`.

**Worktree missing.** `run-metadata.json` contains `worktree_path`. If that
directory no longer exists, re-run `prepare-repo.sh` to recreate it.
