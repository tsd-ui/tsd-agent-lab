# Task: Codebase Map (Read-Only)

Explore this repository and produce a structured codebase map.

## What to include

1. **Directory structure** — Top-level layout with a brief description of each directory's purpose.
2. **Architecture overview** — Key modules, how they connect, entry points, data flow.
3. **Build and test commands** — How to build, test, lint, and run the project.
4. **Dependencies** — Notable dependencies and what they provide.
5. **Risk areas** — Parts of the codebase that look fragile, under-tested, or complex.
6. **Recommended first agent tasks** — Safe, high-value tasks an agent could tackle next (e.g., adding tests, fixing lint warnings, documenting modules).

## Constraints

- **Read-only.** Do not modify, create, or write any files. Do not use the Write or Edit tools.
- Output your findings as your response. The harness captures your output automatically.
- If you cannot determine something, say so rather than guessing.
