# Phase 11—Obsidian team vault setup

This phase sets up a dedicated Obsidian vault for team knowledge sharing around the agent lab.

## Prompt 11A—Create team vault structure

```markdown
Create documentation for setting up a dedicated Obsidian vault for the agent lab team.

Create:
- `docs/obsidian/team-vault-setup.md`
- `docs/obsidian/vault-structure.md`
- `templates/obsidian/vault-template/README.md`

The documentation should cover:
- Why we're using a separate vault from individual work vaults
- Recommended vault location and naming
- Core folder structure:
  - `Skills/` - Documented skills and workflows
  - `Experiments/` - Agent experiment results and learnings
  - `Protocols/` - Communication patterns and harness protocols
  - `Tasks/` - Example tasks and templates
  - `Meeting Notes/` - Team sync notes
  - `Decisions/` - ADRs and key decisions
  - `References/` - External links and resources
- Plugin recommendations (minimal, focused on collaboration)
- Sync strategy (if using Obsidian Sync or git)
- Access control and permissions
- File naming conventions
- Linking conventions

Keep it lightweight. The vault should enhance collaboration without adding overhead.
```

## Prompt 11B—Create team vault templates

```markdown
Create templates for common Obsidian notes in the team vault.

Create:
- `templates/obsidian/skill-documentation.md`
- `templates/obsidian/experiment-log.md`
- `templates/obsidian/task-template.md`
- `templates/obsidian/decision-record.md`
- `templates/obsidian/weekly-sync.md`
- `docs/obsidian/template-guide.md`

Each template should include:
- Frontmatter with relevant metadata
- Clear structure and sections
- Inline guidance on what to include
- Links to related documentation
- Examples where helpful

The templates should encourage:
- Clear documentation of learnings
- Sharing successful patterns
- Recording blockers and solutions
- Building institutional knowledge
- Progressive disclosure (start simple, add detail as needed)

Also document:
- When to use each template
- How to customize for your needs
- How to link related notes
- Best practices for team knowledge sharing
```
